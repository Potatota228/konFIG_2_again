Отчёт по выполнению задания

Тема: Построение графа зависимостей пакетов и реализация базовых операций над ним.

Цель работы

  Целью работы являлась реализация программного модуля, выполняющего построение графа зависимостей программных пакетов (с учётом транзитивности) и поддержку базовых операций анализа графа. Особое внимание уделялось обработке циклических зависимостей и реализации обхода графа без рекурсии.

Ход выполнения работы

  В ходе работы была разработана программа на языке Python, обеспечивающая построение и анализ графа зависимостей пакетов Alpine Linux, а также работу в тестовом режиме с использованием заранее подготовленных файлов-репозиториев. Для реализации использованы только стандартные библиотеки Python (csv, sys, urllib.request, re, ssl), что соответствует условиям задания.

  Программа поддерживает два режима работы:

    режим prod — построение графа реальных зависимостей на основе данных из официального репозитория Alpine Linux;

    режим test — анализ тестового файла, содержащего искусственно заданные зависимости между пакетами, обозначенными заглавными латинскими буквами.

  Основная структура программы включает следующие функциональные блоки:

    Функция main() — основной управляющий модуль, выполняющий чтение и валидацию конфигурации, определение режима работы и запуск соответствующей процедуры построения графа.

    Функции read_config(), validate_config() и print_config() — обеспечивают загрузку, проверку и отображение конфигурационных параметров из файла формата CSV.

    Функции fetch_url() и parse_alpine_page() — реализуют загрузку HTML-страницы пакета и извлечение информации о его зависимостях.

    Функции process_prod_mode() и build_dependency_graph_bfs() — отвечают за построение графа зависимостей пакетов в реальном режиме (prod).

  Алгоритм построения графа реализован с использованием обхода в ширину (BFS) без применения рекурсии.

  На каждом шаге происходит анализ зависимостей текущего пакета и добавление их в очередь на обработку.

  Для предотвращения зацикливания используется множество visited, фиксирующее уже обработанные узлы.

  Функции process_test_mode() и build_test_graph_bfs() — обеспечивают работу программы в тестовом режиме, считывая описание графа из текстового файла, где строки имеют формат

    PACKAGE: DEP1, DEP2, DEP3

  и позволяют моделировать различные сценарии (в том числе с циклическими зависимостями).

  Функция check_cycle() — выполняет проверку наличия циклических зависимостей в графе, предотвращая добавление вершин, приводящих к замыканию.

  Функция print_graph() — выводит итоговую структуру графа зависимостей в читаемом виде, отображая для каждого пакета список всех его прямых зависимостей.

Результаты работы программы

  Программа успешно реализует:

    построение графа зависимостей пакетов с использованием итеративного обхода в ширину (BFS);

    корректную обработку циклических зависимостей с предупреждением пользователя;

    поддержку двух режимов работы: prod (анализ реальных пакетов Alpine Linux) и test (работа с тестовыми файлами);

    отображение полной структуры зависимостей, включая транзитивные связи между пакетами;

    вывод информации в читаемом и наглядном виде.

  В тестовом режиме были продемонстрированы различные сценарии, включая случаи отсутствия зависимостей, линейные цепочки зависимостей и графы с циклическими связями, что подтвердило корректность алгоритма BFS и обработки ошибок.

Программа корректно обрабатывает циклические зависимости, поддерживает как реальный, так и тестовый режим работы, и обеспечивает вывод полной структуры зависимостей для дальнейшего анализа и визуализации.

  Разработанное решение соответствует поставленным требованиям и может быть использовано как основа для дальнейших этапов анализа и визуального представления графа зависимостей пакетов.
